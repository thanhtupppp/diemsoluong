# Runtime Verification

Use this checklist after changing the object detector, decoder, preprocessing,
tracking, or counting behavior.

## Static Checks

```bash
flutter analyze
flutter test --reporter compact
git diff --check
```

## No Active YOLO Runtime References

The current runtime should not reference YOLO model paths or decoder names.
Historical docs are allowed under `docs/legacy_yolov8/` and `docs/superpowers/`.

```bash
rg -n "yolov8|yolov8n_float16|yolov8_int8|decodeDetections|BoxCoordinateFormat" \
  README.md lib test pubspec.yaml docs/model_maker.md
```

Expected result: no matches.

## Model Shape Check

For the bundled model, confirm:

- Input tensor: `[1, 320, 320, 3]`
- Raw boxes: `[1, 19206, 4]`
- Raw class scores: `[1, 19206, 90]`
- Anchor count generated by `buildEfficientDetAnchors(inputSize: 320)`: `19206`

The unit test `test/features/detection/decode_detections_test.dart` verifies
the anchor count, boundary clamping, degenerate-box filtering, optional class
masking, optional score activation, shape-mismatch debug logging, and decoder
adapter behavior.

## Labels Check

The bundled model contains a `labels.txt` metadata file with 90 entries. The
runtime labels in `ModelConfig.cocoLabels` must remain aligned with the sparse
COCO IDs used by the model, including `???` placeholders.

If replacing the model:

- Extract or export the new label file.
- Preserve sparse label IDs if the model uses them.
- Update `ModelConfig.cocoLabels`.
- Add or update decoder tests for the new class count.

## Decoder Configuration Check

The default decoder assumes raw EfficientDet boxes and probability-like class
scores:

- `ScoreActivation.none`
- no `allowedClassIds` filter
- debug shape-mismatch logging only in debug mode from `InferenceIsolate`

If replacing the model:

- Use `ScoreActivation.sigmoid` or `ScoreActivation.softmax` only if the model
  outputs logits.
- Use `allowedClassIds` when the app should count only selected classes.
- Confirm shape-mismatch logs include `boxes.length`, `scores.length`,
  `anchors.length`, `numBoxes`, and `numClasses`.

## Camera And Counting Checks

EfficientDet boxes can differ from the old detector box footprints, even for
the same object. Verify on a physical device:

- Back camera orientation.
- Front camera mirroring.
- Still image overlay alignment.
- Live camera overlay alignment.
- `CountingLine` normalized coordinate mapping.
- Line-cross counting at the intended installation angle.
- `IouTracker` stability under dense scenes and low-FPS jumps.

## Performance Checks

Profile on the lowest target device class:

- Camera frame conversion latency.
- Letterbox preprocessing latency.
- TFLite inference latency.
- Decoder plus NMS latency.
- Memory use during live camera streaming.
- Thermal behavior during at least 5 minutes of continuous scanning.
